Understanding Blazor Server Render Modes

Published: March 19, 2024

Blazor Server in .NET 8 introduces enhanced render modes that give developers more control over how components are rendered and interactive. These render modes are crucial for optimizing performance and user experience.

The Main Render Modes

  • Server: The traditional Blazor Server rendering mode where components are rendered on the server and maintained over a SignalR connection.
  • Static: Components are rendered once on the server with no interactivity. Perfect for static content that doesn't need user interaction.
  • Interactive: Components are initially rendered on the server but then become interactive on the client, maintaining a SignalR connection.

When to Use Each Mode

Choose Server mode when you need real-time updates and don't want to download the entire Blazor runtime to the client. This is great for internal applications or when bandwidth is a concern.

Use Static mode for components that display information but don't need interactivity. This improves performance by reducing the number of SignalR connections.

Pick Interactive mode when you need the best of both worlds - server-side rendering for fast initial page loads and client-side interactivity for responsive user interactions.

Code Example

@page "/counter"
@rendermode InteractiveServer

<PageTitle>Counter</PageTitle>

<h1>Counter</h1>

<p role="status">Current count: @currentCount</p>

<button class="btn btn-primary" @onclick="IncrementCount">Click me</button>

@code {
    private int currentCount = 0;

    private void IncrementCount()
    {
        currentCount++;
    }
}
